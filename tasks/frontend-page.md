# 前台页面模块任务

## 模块目标

用 Next.js/TypeScript 实现关系温度计首版单页体验：重点关系卡片、国家对选择器、关系趋势图、转折点解释、重点报道线索、方法说明和完整状态设计。

## 最小任务

- [x] 建立首版页面入口，默认展示 `chn_usa`。
- [x] 实现页面顶部标题、简短说明、数据更新时间。
- [x] 实现重点关系卡片区：中美、中俄、中欧、中日、中印、美伊、美俄。
- [x] 实现卡片当前温度、3 档文字状态、近 14 日变化、小型趋势线。
- [x] 卡片颜色按 5 档温度分区渲染。
- [x] 点击重点关系卡片后切换主图和解释区。
- [x] 实现两个下拉框组成的国家对选择器。
- [x] 选择器禁止相同对象组合。
- [x] 选择器禁止“欧盟/欧洲”与“英国”“德国”组合。
- [x] 实现 URL 参数读取，支持 `/trend?pair=chn_usa`。
- [x] 当 API 返回规范组合 ID 时，前端将 URL 回写为规范 ID。
- [x] 实现分享按钮，复制当前 URL。
- [x] 实现单对关系趋势图，展示近 90 天 0-100 关系温度。
- [x] 趋势图显示 50 中性线。
- [x] 趋势图标出 Top 3 转折点，移动端可点击，不依赖 hover。
- [x] 点击转折点后展示对应解释摘要、分数变化、解释窗口、前三类驱动事件。
- [x] 实现 3-6 条重点报道线索列表，展示日期、来源域名、URL 线索标题、事件类型、影响方向和原始链接。
- [x] 实现折叠式“数据来源与限制”说明。
- [x] 实现加载态骨架屏。
- [x] 实现超过 8 秒慢加载提示和重试操作。
- [x] 实现 API 错误态，保留页面框架并显示重试按钮。
- [x] 实现 `stale_cache` 或 `stale` 提示：“数据可能不是最新”。
- [x] 实现无数据组合空态。
- [x] 实现数据不足提示：“趋势仅供参考”。
- [x] 实现无明显转折点提示。
- [x] 实现候选池少于 2 个对象时禁用选择器。
- [x] 移动端使用纵向布局，重点卡片横向滑动，图表全宽，解释区在图表下方。
- [x] 桌面端使用卡片网格，趋势图与解释区左右分栏。
- [x] 趋势图和解释区使用同一组件树，通过 CSS media query 或 container query 切换布局。
- [x] 编写前端测试，覆盖卡片点击、选择器、URL 分享、状态展示、移动端转折点点击、响应式布局。

## 完成定义

- [x] 用户可以选择任意合法候选组合并查看趋势。
- [x] 用户可以点击转折点并看到原因解释和重点报道线索。
- [x] 手机端和桌面端都能完整完成核心路径。
- [x] 页面不直接访问数据库，不生成虚假新闻标题。
