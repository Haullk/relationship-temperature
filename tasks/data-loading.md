# 数据装载模块任务

## 模块目标

用 Python 从本地 PostgreSQL 读取候选关系所需的 GDELT clean 事件，输出标准化事件列表。该模块只做配置展开、查询和轻量标准化，不计算关系温度，不做解释层噪音过滤。

## 最小任务

- [x] 建立 Python 运行入口，读取数据库连接和候选池配置。
- [x] 新建或整理 `config/candidate-pool.json` 的首版配置格式。
- [x] 在配置中写入候选对象：`CHN`、`USA`、`RUS`、`JPN`、`IND`、`IRN`、`TWN`、`GBR`、`DEU`、`europe`。
- [x] 在配置中写入欧洲聚合口径：`EUR + GBR + DEU + FRA + ITA + ESP + NLD`。
- [x] 生成候选池合法组合列表，排除同一对象组合。
- [x] 生成合法组合时排除 `europe_gbr` 和 `europe_deu`。
- [x] 为每个合法组合生成规范组合 ID，按两个对象代码字母序排序后用 `_` 连接。
- [x] 实现 `chn_usa` 这类普通双边组合的双向事件查询。
- [x] 实现 `chn_europe` 这类聚合对象组合的展开查询。
- [x] 查询默认近 90 天数据，并保留实际数据覆盖范围。
- [x] 过滤缺少日期、双方代码或 `goldstein_scale` 的明显无效记录。
- [x] 输出标准化事件字段：日期、对象 A、对象 B、Goldstein、mentions、articles、事件类型、actor 名称、来源域名、来源 URL。
- [x] 对无数据组合返回空事件列表，不抛出不可恢复错误。
- [x] 编写模块测试，覆盖普通组合、欧洲聚合、欧洲重叠组合排除、空数据组合。

## 完成定义

- [x] 数据处理模块可以直接消费标准化事件列表。
- [x] `europe_gbr` 和 `europe_deu` 不会进入合法组合列表。
- [x] 数据装载模块不包含关系温度计算逻辑。
- [x] 数据装载模块不包含解释层文本相关性过滤逻辑。
