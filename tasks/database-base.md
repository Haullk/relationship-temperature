# 数据库基础模块任务

## 模块目标

为关系温度计提供只读 MapNews GDELT clean 数据访问，以及关系产品专用缓存存储。不得修改 MapNews 项目文件夹，不得破坏 MapNews 既有表结构和导入流程。

## 最小任务

- [x] 确认本项目使用的数据库连接配置来源，不在文档或代码中暴露真实密码。
- [x] 只读检查 `gdelt_events_clean` 表是否存在。
- [x] 只读检查 `gdelt_events_clean` 是否包含事件日期、双方 actor 国家代码、`goldstein_scale`、`num_mentions`、`num_articles`、事件类型、actor 名称、来源域名、来源 URL 等字段。
- [x] 记录实际字段名与产品字段名的映射，供数据装载模块使用。
- [x] 确认当前数据库可用数据覆盖范围，至少能返回近 90 天窗口。
- [x] 设计关系产品缓存表结构，至少支持关系组合 ID、对象 A/B、数据范围、生成时间、趋势点、转折点、解释、缓存状态。
- [x] 设计缓存写入原子性策略，三选一并写入实现说明：全量事务写入、staging 表再 swap、或接受逐条写入的短暂不一致。
- [x] 编写新增缓存表的 migration，不修改 MapNews 既有表。
- [x] 如查询性能不足，编写必要索引 migration，确保索引不改变原表字段含义。
- [x] 编写缓存表创建和清理的本地验证命令。
- [x] 编写数据库模块测试或检查脚本，验证缓存表可写、GDELT clean 表可读。
- [x] 验证新增表和索引后，MapNews 原有表仍可正常查询。

## 完成定义

- [x] 数据库中存在关系产品独立缓存结构。
- [x] MapNews 既有表结构和项目文件夹未被修改。
- [x] 数据装载模块能基于字段映射读取所需事件字段。
- [x] 服务 API 能从缓存表读取生成时间、数据范围和关系详情。
