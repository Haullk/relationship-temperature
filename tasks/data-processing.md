# 数据处理模块任务

## 模块目标

用 Python 将标准化事件列表转换为可展示的关系趋势缓存：关系温度、14 日平滑、近 14 日变化、Top 3 转折点、解释摘要、驱动事件和重点报道线索。

## 最小任务

- [x] 建立数据处理模块入口，接收数据装载模块输出的标准化事件列表。
- [x] 实现事件权重：`event_weight = ln(1 + max(num_mentions, num_articles, 1))`。
- [x] 实现每日报道加权 Goldstein：`sum(goldstein_scale * event_weight) / sum(event_weight)`。
- [x] 实现 14 日滚动平均：当前日及前 13 日。
- [x] 实现关系温度映射：`clamp(50 + rolling_14d_goldstein * 5, 0, 100)`。
- [x] 实现 5 档温度分区：明显偏冲突、偏冲突、接近中性、偏合作、明显偏合作。
- [x] 实现卡片 3 档状态折叠：合作、中性、冲突。
- [x] 实现近 14 日变化：`delta_14d >= 3` 为改善，`<= -3` 为恶化，否则平稳。
- [x] 实现转折点候选：当前 14 日平滑温度减去 14 天前温度。
- [x] 实现 Top 3 转折点筛选，最小间隔 14 天。
- [x] 实现转折点状态原因码：`normal`、`data_insufficient`、`no_significant_turning_points`。
- [x] 实现解释窗口：变化期 `t - 6` 至 `t`，基准期 `t - 13` 至 `t - 7`。
- [x] 实现驱动事件统计，对 actor 名称、来源域名、来源 URL 做文本相关性过滤。
- [x] 实现重点报道线索筛选，对来源域名或来源 URL 做更严格相关性过滤。
- [x] 实现重点报道排序，综合方向一致性、Goldstein 强度、报道量和 URL 可读性。
- [x] 实现一句话模板解释，不生成确定因果，不生成虚假标题。
- [x] 生成可写入缓存表的关系结果对象。
- [x] 编写模块测试，覆盖权重、温度映射、14 日平滑、变化阈值、转折点间隔、原因码、解释窗口和文本过滤。

## 完成定义

- [x] 同一输入事件列表能得到稳定、可重复的趋势结果。
- [x] 所有输出温度都在 0-100 范围内。
- [x] 数据不足 30 天时返回 `data_insufficient`。
- [x] 有足够数据但无明显转折点时返回 `no_significant_turning_points`。
- [x] 重点报道只使用本地数据库已有字段。
