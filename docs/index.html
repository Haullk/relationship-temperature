Architecture

GeoPrizm is split into a Python data pipeline, PostgreSQL relationship cache, and a Next.js application that serves both the dashboard and internal JSON endpoints.

1. GDELT events Fetch or read GDELT 2.0 event exports for candidate country pairs.
2. Normalize Filter actors, map country objects, and store compatible event rows.
3. Score Aggregate GoldsteinScale cooperation/conflict signals into daily indexes.
4. Detect shifts Find visible relationship changes and select supporting report signals.
5. Enrich Generate AI-assisted explanations from structured signals and metadata.
6. Serve Read precomputed cache from Next.js UI and JSON endpoints.

The endpoints below are used by the current web app. Treat them as internal, experimental APIs until a versioned public API is added.

AI explanation refresh POST 
POST /api/ai/explanation
Content-Type: application/json

{

Local Development

The project combines Node.js for the web application and Python for data processing. Secrets belong in .env.local and should never be committed.

Install and run 
npm install
python3 -m venv .venv
.venv/bin/python -m pip install -e ".[dev]"
cp .env.example .env.local

npm run dev -- -p 3001
Quality checks 
npm run typecheck
npm run lint
npm test
npm run build

.venv/bin/python -m pytest
.venv/bin/python -m mypy relationship_temperature tests
.venv/bin/python -m ruff check relationship_temperature tests

Data Refresh

Relationship products are precomputed before the dashboard reads them. This keeps the web app fast and makes trend cards, charts, turning points, and explanations reproducible.

Standalone GDELT import and cache refresh 
.venv/bin/python -m relationship_temperature.gdelt_importer \
  --date 2026-06-02 \
  --wait-for-files \
  --precompute \
  --with-ai \
  --prune-days 120
